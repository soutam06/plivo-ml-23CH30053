LLM 2000-Step Speedrun Summary

This report details the architectural choices, optimization strategies, and iterative process used to minimize Bits Per Byte (BPB) on the provided GPT model within the strict 2M parameter and 2,000 step budget limits.

Agent vs. Human Collaboration:
This submission was highly collaborative. The Human Developer (User) handled environment configuration, WSL interactions, version control management, and the execution of the training loops. The AI Coding Agent researched the "nanoGPT speedrun" state-of-the-art, hypothesized architectural upgrades, wrote the PyTorch implementations for RMSNorm, SwiGLU, and QK-Norm, and tuned the hyperparameter schedule. Together, this iterative feedback loop drastically reduced the baseline BPB.

1. Final Model Architecture

The baseline model was a deliberately mediocre 1.3M parameter GPT. We upgraded it to a ~1.83M parameter modern architecture that takes inspiration from LLaMA and the nanoGPT speedrun community.

  • Parameter Scaling: We increased the embedding dimension (n_embd) from 160 to 192, and the number of attention heads (n_head) from 4 to 6. We kept 4 layers to prioritize width over depth, preventing gradient instability.
  • Weight Tying: We set tie_weights = True to share the token embeddings with the output linear head. This freed up thousands of parameters, allowing us to build a wider model within the 2M limit.
  • RMSNorm: We replaced standard LayerNorm with RMSNorm, which removes mean-centering and reduces computational overhead while preserving training stability.
  • SwiGLU Activation: We replaced the standard GELU Multi-Layer Perceptron (MLP) with a Gated Linear Unit (SwiGLU). SwiGLU creates sparser activations and yields superior performance per parameter.
  • QK-Norm: We applied RMSNorm to the Queries and Keys inside the Self-Attention mechanism. This critical speedrun trick prevents attention logits from growing out of bounds, allowing us to safely push the learning rate much higher.

2. Optimizer & Training Regimen

To maximize learning in exactly 2,000 steps, the training script was overhauled:

  • AdamW Optimizer: Replaced standard Adam with AdamW, utilizing a weight_decay of 0.01 for better regularization.
  • OneCycleLR Scheduler: Replaced the constant learning rate with a dynamic schedule. The LR warms up to a peak of 1e-3 for the first 10% of steps, then cosine-decays down. This enables the model to rapidly traverse the loss landscape early on and finely converge at the end.
  • Gradient Clipping: Implemented a max norm of 1.0 to prevent exploding gradients at the higher peak learning rate.

Why This Strategy is Optimal

In a heavily constrained environment (120 minutes wall-clock, CPU-only, 2000 steps), the primary bottleneck is how much information the model can absorb per step. By combining QK-Norm and a OneCycleLR scheduler, we artificially forced the model to learn at an aggressively high learning rate without crashing. By using SwiGLU and Tied Weights, we ensured that every single one of the 1.83 million parameters was maximally expressive. This methodology directly aligns with the latest research in the open-source LLM speedrun community, making it the best possible metric optimization approach under these constraints.
